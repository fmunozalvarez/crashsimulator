CrashSimulator Private Labs Workflow

This page explains how to work privately on CrashSimulator improvements while keeping the public community repository clean, intentional, and safe.

Repository Model

RepositoryVisibilityPurpose
fmunozalvarez/crashsimulatorPublicStable community code, sanitized documentation, public releases, issues, and pull requests.
fmunozalvarez/crashsimulator-labsPrivateExperimental community features, prototypes, private validation, roadmap work, and pre-release evidence review.
fmunozalvarez/crashsimulator-enterprisePrivate future optionEnterprise-only controller, dashboards, agents, commercial packaging, connectors, and license-sensitive integrations.

Create Or Verify The Labs Repository

Create fmunozalvarez/crashsimulator-labs as a normal private repository, not as a fork. Keep it empty before the first push.

Local Setup Commands

mkdir -p /Users/franciscomunozalvarez/Downloads/CrashSimulator_Labs
git clone https://github.com/fmunozalvarez/crashsimulator.git \
  /Users/franciscomunozalvarez/Downloads/CrashSimulator_Labs/source
cd /Users/franciscomunozalvarez/Downloads/CrashSimulator_Labs/source

git remote rename origin community
git remote set-url --push community DISABLED
git remote add origin https://github.com/fmunozalvarez/crashsimulator-labs.git
git remote -v

git switch main
git push -u origin main
git push origin --tags

Expected Remotes

community  https://github.com/fmunozalvarez/crashsimulator.git (fetch)
community  DISABLED (push)
origin     https://github.com/fmunozalvarez/crashsimulator-labs.git (fetch)
origin     https://github.com/fmunozalvarez/crashsimulator-labs.git (push)

Branch Strategy

Branch patternLocationPurpose
mainPrivate LabsMirror public main unless private baseline changes are intentional.
labs/<feature>Private LabsExperimental work and prototypes.
publish/<feature>Local onlyClean staging branch used to prepare selected public contributions.
feature/<feature>Public repo onlyPublic pull-request branch after private work is cleaned.

Daily Workflow

cd /Users/franciscomunozalvarez/Downloads/CrashSimulator_Labs/source
git switch main
git fetch community
git merge --ff-only community/main
git push origin main

git switch -c labs/dbsat-import-engine
git add .
git commit -m "Prototype DBSAT report import engine"
git push -u origin labs/dbsat-import-engine

Publish Clean Work To Public

git fetch community
git switch -c publish/dbsat-docs community/main
git cherry-pick <safe_commit_sha>
git diff --stat community/main..HEAD
git diff community/main..HEAD
git grep -nE "password|secret|token|BEGIN PRIVATE KEY|wallet|ocid1|credential" -- .
git push https://github.com/fmunozalvarez/crashsimulator.git \
  HEAD:feature/dbsat-import-foundation

Never Commit

  • ADB wallets, TDE wallets, OCI API keys, SSH private keys, or RMAN credentials.
  • DBSAT customer reports, customer topology reports, real audit evidence, or logs.
  • Private roadmap, customer screenshots, or enterprise-only code into the public repo.

Commands To Avoid From Labs

git push --all community
git push --mirror community
git push community --tags
git push community labs/*

Operating Rules

  1. Labs can be messy; public cannot.
  2. Labs branches never go directly public.
  3. Public changes are cherry-picked or squash-merged.
  4. Real wallets, reports, audit evidence, logs, and customer data never go public.
  5. Public releases are built only from public main or public release branches.
